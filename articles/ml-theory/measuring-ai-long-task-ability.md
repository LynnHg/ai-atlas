# 衡量 AI 完成长任务的能力：指数增长趋势与预测

> 原文链接：[Measuring AI Ability to Complete Long Tasks](https://metr.org/blog/2025-03-19-measuring-ai-ability-to-complete-long-tasks/)
>
> 日期：2025-03-19
>
> 作者：Thomas Kwa, Ben West, Joel Becker, Amy Deng 等（METR）
>
> 主题：evaluation, agent, capability-trend, benchmark
>
> 类型：article

---

## 一、核心发现

用**人类完成该任务需要多长时间**来衡量 AI Agent 能力。

当前前沿模型的表现：

| 人类任务时长 | 模型成功率 |
|-------------|-----------|
| < 4 分钟 | ~**100%** |
| ~1 小时 | ~**50%**（Claude 3.7 Sonnet） |
| > 4 小时 | **< 10%** |

> 这解释了 AI 在基准测试上超越人类、但在日常工作中无法可靠替代人类的矛盾：日常工作通常是更长时间的任务。

---

## 二、指数增长趋势

| 指标 | 数值 |
|------|------|
| 趋势类型 | 指数增长 |
| 翻倍时间 | 约 **7 个月** |
| 趋势持续时间 | 过去 **6 年**一致 |
| SWE-Bench 上的翻倍时间 | < **3 个月** |

预测：
- 如果趋势持续 **2-4 年** → Agent 能完成**持续一周**的任务
- 如果趋势持续到**本十年末** → Agent 能自主执行**持续一个月**的项目

---

## 三、方法论

### 度量方法

1. 让人类专家完成多步骤软件和推理任务，记录完成时间
2. 对每个模型，用人类完成时间拟合逻辑回归曲线预测模型成功率
3. 在固定成功率（如 50%）处读取对应的任务时长 = 模型的"时间范围"

### 为什么用"任务时长"而非传统基准

| 传统基准（正确率） | 任务时长 |
|-------------------|---------|
| 只覆盖窄范围难度 | 覆盖跨多个数量级的能力水平 |
| 难以对应真实世界影响 | 直接关联真实世界影响 |
| 百分比分数不可靠（难度分布不均） | 既能比较相对性能，也能解释绝对能力 |

### 鲁棒性

- 在多种任务子集上趋势一致
- 在 SWE-Bench Verified 独立数据集上复现
- 即使绝对测量偏差 **10x**，也只影响预测到达时间约 **2 年**

---

## 四、为什么 Agent 在长任务上失败

> AI Agent 更多是在**串联更长的动作序列**时遇到困难，而非缺少单步所需的技能或知识。

关联因素：
- 上下文窗口有限（Context Rot）
- 错误会复合（级联失败）
- 跨上下文窗口的记忆断裂

---

## 五、核心启示

1. **"任务时长"是衡量 Agent 能力最实用的维度**——比正确率更能预测真实世界影响
2. **当前最强模型可靠完成约几分钟的任务**——虽然偶尔能处理数小时的任务
3. **能力翻倍时间约 7 个月**——过去 6 年一致的指数趋势
4. **2-4 年内可能出现完成一周任务的 Agent**——如果趋势持续
5. **Agent 的瓶颈是串联长序列动作，而非单步能力**——与上下文工程的重要性直接对应
6. **测量误差对预测影响有限**——10x 误差仅影响 ~2 年到达时间
7. **这个指标对 Agent 工程有实际指导意义**——知道模型能可靠处理多长的任务，才能合理设计架构
