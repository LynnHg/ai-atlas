# 学习文章并记录笔记

当用户提供一个文章 URL 并表达学习/记录意图时，执行以下完整流程。

## 工作流程

### 第一步：抓取文章

用 WebFetch 获取文章内容。如果失败，用 WebSearch 搜索文章标题后重试。

### 第二步：分析并向用户展示学习要点

用中文向用户展示从文章中能学到什么，要求：

- 按主题分章节整理，使用清晰的层级标题
- 提炼核心观点、关键数据、实用技巧
- 用表格对比不同方案/概念的差异
- 标注关键数字和数据（加粗）
- 在末尾给出"核心启示"总结列表

### 第三步：确定主题分类

根据文章内容，匹配到以下主题目录之一：

| 主题目录 | 适用内容 |
|----------|----------|
| `agent` | AI Agent、多 Agent、编排模式、工具调用 |
| `prompt-engineering` | Prompt 工程、系统提示词、上下文工程 |
| `rag` | 检索增强生成、向量数据库、分块策略 |
| `mcp` | MCP 协议、Server 开发、工具集成 |
| `fine-tuning` | 模型微调、LoRA、数据集 |
| `app-dev` | AI 应用开发、API 集成、生产实践 |
| `ml-theory` | Transformer、注意力机制、训练原理 |

如果文章跨多个主题，放在**最核心**的主题目录下。

### 第四步：保存笔记文件

在 `articles/<主题>/` 下创建 md 文件。

文件名：用英文短横线命名，概括文章核心内容（如 `building-effective-agents.md`）。

文件头部**必须**包含元信息：

```markdown
# 中文标题

> 原文链接：[英文原标题](URL)
>
> 日期：YYYY-MM-DD（文章发布日期）
>
> 作者：作者名
>
> 主题：tag1, tag2, tag3
>
> 类型：article
```

文件正文：即第二步中展示给用户的完整内容，末尾附上文中提到的参考资源链接。

### 第五步：更新索引

1. 更新 `articles/README.md` 的文章索引表，按日期**降序**插入新行：

```markdown
| 发布日期 | 主题 | 标题 | 来源 |
|----------|------|------|------|
| YYYY-MM-DD | 主题 | [中文标题](主题/文件名.md) | 来源名称 |
```

2. 更新根 `README.md` 的"最近更新"表，按日期**降序**插入新行：

```markdown
| 发布日期 | 类型 | 主题 | 标题 |
|----------|------|------|------|
| YYYY-MM-DD | article | 主题 | [中文标题](articles/主题/文件名.md) |
```

### 第六步：确认完成

告知用户文件保存路径和更新的索引。
